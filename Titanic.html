4. Formulando a pergunta¶

Aps conhecer e entender a estrutura dos nossos dados, assim como os seus valores, a dvida que mais se acentua :

Quais features(colunas) tem uma correlao com o fator sobrevivncia?

Como em todo o desastre, preferncias de sobrevivncia so dadas moralmente para mulheres, crianas, idosos e familias, porm tambm sabemos que pessoas de alto poder aquisitivo podem ter sido beneficiadas.

Ento vamos analisar e descobrir se os fatores como Taxa(FareGroup),Classe do Ticket(Pclass), Genro(Sex), Famlia/Sozinho(FamilySize) afetaram o fator sobrevivncia tambm no desastre do Titanic.

As perguntas seram as seguintes:

  • O fator classe ou fator taxa da passagem tiveram relao com o fator sobrevivncia? Pessoas com poder aquisitivo maior tinham propenso a sobreviver mais que pessoas com poder aquisitivo menor?
  • O fator ser passageiro nico ou viajar em famlia teve relao com o fator sobrevivncia? Famlias tiveram mais chances de sobreviver do que pessoas viajando sozinha? E o tamanho da famlia interferia nessa correlao?
  • O fator genro teve relao com o fator sobrevivncia? Mulheres tiveram mais propenso a sobreviver do que os homens?

5. Classificando as colunas do dataframe¶

Dados categóricos¶

So dados que servem de label de um grupo de items ou indivduos. Dados categricos possuem a seguinte subdiviso:

  • Ordinal: So dados que no ordem ou algum tipo de ranking de classificao associado.
  • Nominal: So dados que possuem um ranking de classificao associado.
Dados Quantitativos¶

So dados na qual podemos operar matematicamente em cima deles para obter insights teis. Dados quantitativos possuem a seguinte subdiviso:

  • Discreto: So dados que podem ser subdivididos cada vez mais em menores unidades.
  • Contnuo: So dados que no podem ser subdividios em menores unidades.

Nossa classificao das colunas a seguinte:

Dados categóricos¶
  • Ordinal: Pclass
  • Nominal: Survived, Sex, and Embarked
Dados Quantitativos¶
  • Discreto: SibSp, Parch
  • Contnuo: Age, Fare

7. Estratégia para as colunas com dados faltantes¶

7.1 Dados Quantitativos¶

Iremos complementar as colunas com dados faltantes utilizando a mediana pra cada Title e transformar o tipo da coluna pra inteiro, que a representao adequada para a coluna. Os seguintes passos sero executados:

  1. Criaremos uma coluna chamada Title para nos ajudar a calcular a idade referente ao ttulo que a pessoa possui no nome
  2. Calcularemos a mediana da idade pra cada ttulo
  3. Preencheremos os valores nulos com a mediana da idade referente ao ttulo que a pessoa possui
In [620]:
passengers["Title"] = passengers.Name.str.extract('([A-Za-z]+)\.', expand=False)
ages                = passengers.groupby('Title')['Age'].median()
for title in passengers.Title.unique():
    median = ages[title]
    passengers.loc[(passengers.Age.isnull()) & (passengers.Title == title),"Age"] = median

9. Visualização Features x Sobrevivência¶

Primeiro construmos uma funo que passado o nome da feature, ela gera um grfico relacionado a feature Survived

def generate_graph(feature,graph="bar"):
    """
    Funo que recebe o nome da feature e gera o grfico relacionado a feature Survived
    Argumentos:
        feature:  String contendo o nome da feature
    Retorna:
        None
    """
    ax       = None
    group    = passengers.groupby(["Survived",feature],sort=True).size()
    df_aux   = pd.concat([group[0],group[1]],axis=1).fillna(0)
    ax       = df_aux.plot(kind=graph,title=feature,grid=True,legend="reverse",rot=60,stacked=False)
    ax.set_ylabel("Number of people")
    ax.legend(["Not Survived", "Survived"]);
